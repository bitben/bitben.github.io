How to install git on shared hosting

posted on: 11-08-2013 10:20 AM / written by: bensonshaji / For: linux

Popularity

Tweet

As we all know that Git has become the most popular version control system in the recent years, the need of it is also increasing among programmers and developers. If you have a virtual server or using amazon cloud then you don’t have to worry about installing Git. Just follow general procedures as per your linux distribution for installing softwares. But what if your site doesn’t need a virtual private server or amazon web service? what if you cannot spend a hundred bucks for virtual private server? This is the situation where you tend to purchase a reasonable hosting plan for the budget you have. But these shared hosting will not give you full control over SSH or any other linux features. Most of the people dont know that shared hosting server can also be tweaked in many ways to install softwares that are not officially supported by them. This tweak is completely legal providing that the software you are about to install has a GNU or any other open source licence. Yes ofcourse Git is Open Source!

In this article I am using my Godaddy’s Ultimate account as a target for installing the Git.

Before starting there are certain requirements that you have to keep in mind for this tweak to work.

  • Web hosting server
  • Secure Shell – SSH
  • Rpm of Git that works under your server

If were are not clear with the third point, it means: suppose if your server is using Red-hat-6 as its operating system then you need to find out what version of Git does it supports or that is available in Red hat’s repository.

So lets begin:

My hosting server uses Cent OS 5.8 (final) as its operating system, so this might also work on Red hat 5.8

Step 1 Login to your account and go to the root directory / or SSH to your account

$ssh bitben.org
$cd

Step 2 Download the Git rpm for the operating system supported by your hosting server. Since my hosting server is Godaddy and uses Cent OS 5.8 (final) as its operating system, it supports Git 1.8.2.1-1 as of the date 11/08/2013. When your server’s operating system is upgraded you might want to repeat this tutorial for the latest version of Git. Make sure that these commands are executed in the root directory so that you can keep you website codes clean from the software codes.

$mkdir git
$wget http://dl.fedoraproject.org/pub/epel/5/i386/git-1.8.2.1-1.el5.i386.rpm
$rpm2cpio git-1.8.2.1-1.el5.i386.rpm | cpio -imdv

Now you can remove the downloaded file as you have extracted the git rpm. Let me explain what does the above code means. You are using wget to download the git rpm, then rpm2cpio is small tool used to extract rpm package manager file into cpio archive. More information about cpio archive can be found here Wikipedia cpio. Now you can see two folder created in the git directory i,e /etc and /usr. Step 3 Now that you have the executable files in the /usr/bin folder you can use it directly by providing specific path to the git file. but lets make it user friendly and more clean via .bashrc. create a new hidden file called .bashrc if it doesn’t exist and enter the following lines

GIT Configuration
export GIT_BIN=${HOME}/git
export PATH=${GIT_BIN}/usr/bin:${PATH}
export GIT_EXEC_PATH=${GIT_BIN}/usr/bin
export GIT_TEMPLATE_DIR=${GIT_BIN}/usr/share/git-core/templates
export GIT_SSL_NO_VERIFY=true

Make the last line as false if your hosting server support SSL. Once you have added the following lines into your .bashrc file, hit save and close it.

Now that you have saved it. Git can be easily accessed once your bashrc is activated. After SSH ing to your account use the command to activate the .bashrc file where Git configuration is added.

$source .bashrc

This command tells bash to use the configuration from the file .bashrc. Now type

$git --version

to check whether the git is working. If you get an output as

git version 1.8.2.1

Then you have succesfully installed Git on your shared hosting account. What next? Read this article to make use of git for your next web project Git for web developers

Note: If you have succesfully installed Git on a shared hosting server apart from Godaddy, kindly provide the details of the (git version you used + operating system and version + Hosting provider + URL of the git software) through comments, so that noobs can also make use of this article.

For example:

Godaddy + Cent OS 5.8 + Git-1.8.2.1 + http://dl.fedoraproject.org/pub/epel/5/i386/git-1.8.2.1-1.el5.i386.rpm
