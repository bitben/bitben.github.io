So finally we are ready to continue with the second part of the tutorial, on how to generate static blog using python pelican. In the previous article Python Static Blog Generator Part 1 we discussed and learnt about setting up pelican on a linux system. In this tutorial we will show you how to create a custom pelican theme which is responsive in nature with loads of features added to the theme that is required by every blogger. So lets get started.

By default pelican uses “notmyidea” theme. We will take this theme as an example to build our new theme. I have used bootstrap 3 for my site bitben.org. So I will be considering to use boilerplate for this tutorial. At first let us analyze the structure of our default theme. If you have installed pelican on a virtual environment then the path for your default theme will be “{your directory where virtual python is installed}/lib/python2.7/site-packages/pelican/themes/notmyidea/”. Now you can see the structure of default theme directory.

NOTMYIDEA-THEME-DIRECTORY-STRUCTURE
	|-> static
	|--------> css
	|----------------------> main.css
	|--------> images
	|----------------------> Example.jpg
	|--------> javascript
	|----------------------> scripts.js
	|--------> morefolders
	|----------------------> More

	|-> templates
	|--------> index.html
	|--------> base.html
	|--------> article.html
	|--------> archives.html
	|--------> author.html
	|--------> Much more....

So this will be the basic structure of directories and files arrangement of the default pelican theme which we have replicate for our theme. Now let us analyze how the boilerplate template directory is structured.

BOILERPLATE Directory
	|-> CSS
	|--------> main.css
	|--------> normalize.css
	|--------> normalize.min.css

	|-> IMG
	|--------> your images

	|-> JS
	|--------> main.js
	|--------> vendor
	|--------------------> subfiles

Copy paste the files from boilerplate to the new theme directory accordingly and update the paths in index.html file. Now let us modify our static html files from the new theme directory into dynamic one. Once you have created the theme structure and linked the paths properly, we need to start editing the files in the template folder.
