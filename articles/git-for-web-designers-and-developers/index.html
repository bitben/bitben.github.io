Git for web designers and developers

posted on: 06-06-2013 10:10 AM / written by: bensonshaji / For: developers

Popularity

Tweet

In the recent years git has become the most popular version control system used by a wide group of software developers. Its application is not only limited to the software developers but can also be used by web designers, developers, graphic designers, blog authors etc… Generally saying it can be used by all those who wants to have a look back into the past to change the future.

Git was founded by Linus Torvalds, Yes the same guy who founded linux operating system. This software was created with an intension to maintain different revisions of codes inorder to test and develop any software application (In specific for linux kernel development). The main advantage of this software is that you can switch back to the previously executed set of codes or the codes that worked at certain stage. You can link tiny messages during each modification of the code inorder to remember what you had coded in the past.

Since it was founded mainly to serve the purpose of software developers, they are the vast group who are using it more than any other in the geek world.

As the title of this article says as “How git can be usefull for web designers or web developers”, I am going to discuss in todays article about the usage of git as a web designer or developer.

Web developers are not software developers but that doesn’t mean they work with codes; ofcourse they do work with codes, graphics, text, etc… While developing a website or web application there are few points that has to be considered such as “Browser compatibility, user interface & contents” These three are the major factors that should be considered before developing any websites or web application.

Even though there are hundreds of browsers out there; Internet Explorer, Mozilla Firefox, Google Chrome and Safari are the popular ones used by many. Even though in recent days browsers are improving a lot with great features and user friendliness there are certain categories where a web designer or developer need to interfere in order to make sure that the feel of websites or web application looks same in all browsers and operating systems.

Let us consider an example of a css property i,e “border-radius”

I have written a small piece of code to display a square in the web page and the css code is as follows:

.square {
    width: 100px;
    height: 100px;
    background-color: Red;
    border-radius: 2px 2px 2px 2px;
    }

Now, when I run the code in the browser we can see that a square is displayed and border radius of 2px is applied for it in google chrome browser. But it doesn’t look the same in all browser. Now I need to update the code which can support many browsers. So I am updating the code as follows.

.square {
    width:100px;
    height:100px;
    background-color:Red;
    border-radius: 2px 2px 2px 2px;
    -webkit-border-radius: 2px 2px 2px 2px;
    -mozilla-border-radius: 2px 2px 2px 2px;
    }

Now the border radius will be applied for the browsers that support webkit property such as safari and chrome. I dont know whether you have noticed or not that I have purposefully made a mistake for the sake of this article i,e -mozilla-border-radius is a wrong syntax. Suppose after making this mistake you have proceeded with 200 lines of code and noticed that there are many errors inlcluding the above mentioned one.

This is where you think for going back to the previously working stage where the codes were working fine. And this is where you need version controlling inorder to roll back to the previous stage. If you were been using git then you could have roll back to the commit which was working well with one command as follows.

$git reset hard commitnumber

After all this lengthy story as an example lets begin with the actual article on using git as a web designer or developer.

Let us assume that “myproject” is the directory where you are working with the web project. Initiating the git repository is as easy as follows

$cd myproject/
$git init

You are done with initiating a git repository.

Now lets add all the files which needs to be controlled by the version control system git

$git add *  or  $git add file1 file2 file3

Now that you have included files to the git repository, lets commit it with a message since all codes and files are setup properly.

$git commit -m "git repository initiated and setup"

Now you have made your first commit.

Continue the same procedures after making certain set of stages and commits. Now that you have made 100 commits and are not satisfied with the 100th commit and you want to roll back to the 95th commit. Then run the command as follows.

$git reset --hard 95thcommitnumber

where commit number can be obtained using

$git log

Once when the project is finished you can upload all your files except the .git folder to your web server. Its not mandatory that you dont have to upload, just to save space and bandwidth you can avoid the following.
