Version control for website via ftp with bazaar

posted on: 25-08-2013 10:10 AM / written by: bensonshaji / For: developers

Popularity

Tweet

Version control system can play a major role in day to day life of a designer or a developer. With version control system you have the full power to switch between different versions of the code/files or anything you want. Git, Subversion, Mercurial, Bazaar are the best in this field. Each one has its own pro’s and con’s, which I will not be explaining in this article. Bazaar is a version control system provided by the canonical team. This article targets using bazaar with ftp for version controlling. Developers who don’t have full control over their server via ssh can make use of this tutorial to utilize the power of version control. Why Bazaar? because, bazaar is the only version control system that provides native support for different protocols such http, https, ftp, sftp etc..

This article is divided into 2 sections

  • Setting up bazaar and plugins
  • Creating and deploying bazaar repository

Setting up bazaar and plugins

Lets begin with installing bazaar on linux.

Ubuntu users

$sudo apt-get install bzr
# This will install bazaar and its dependencies

Fedora users

$su -c "yum install bzr"
# This will install bazaar and its dependencies

At this stage bazaar is ready to be served. But we also need a plugin which can upload and delete files from your repository via ftp. Setting up this plugin is very simple with few commands as follows.

Install bzr upload plugin

$mkdir ~/.bazaar/plugins
$cd ~/.bazaar/plugins
$bzr branch https://launchpad.net/bzr-upload
$cd bzr-upload
$python setup.py build
$sudo python setup.py install

With these commands executed, you have successfully installed bazaar plugin to transfer files via ftp.

Creating and Deploying bazaar repository via ftp

Let us initiate a bazaar repository on the ftp server. You will have to put your ftp details to initiate a bazaar repository on your server.

$bzr init ftp://username@ftpserver.com/yourfolder
# Enter your ftp password and wait for a while so that .bzr folder is created in the mentioned ftp path. 

$bzr checkout ftp://username@ftpserver.com/yourfolder
# Now you are cloning the created repository from the ftp server. This is similar to clone command in Git. 

$cd yourfolder
# Goto the folder which was created after executing bzr checkout command.

$touch index.html style.css script.js
# Create some files and edit them for committing.

$bzr status
# Check the status of bazaar repository to commit any changes

$bzr add
# Add all files in the current folder for committing

$bzr commit -m "Bazaar repository initialized via ftp"
# You are creating first revision of your files with revision title as mentioned above
# Enter the ftp password

$bzr upload ftp://username@ftpserver.com/yourfolder
# Upload the files via ftp. Now you have your site running on the mentioned path

$bzr rm script.js
# Delete file called script.js

$bzr commit -m "I just deleted script.js as it had lot of bugs"
# This is the second revision with a file deleted from bazaar repository.
# Enter the ftp password

$bzr upload ftp://username@ftpserver.com/yourfolder
# This deletes the files script.js from your ftp server

So in this way you can keep versions of your project on your local system and upload and delete changes from your ftp server by using bazaar from the command line. Kindly give your feedback and suggestion on this article.
