How to secure git repository on shared hosting

posted on: 12-08-2013 10:10 AM / written by: bensonshaji / For: linux

Popularity

Tweet
Yesterday I had posted an article about installing git on shared hosting. I got a very good feedback from many people on google plus and via mail. Some had quesions about the security of the git repository on a web server. I am not sure whether most of you guys know that .git folder of a git repository can be accessed by anyone if not configured properly. All the source files of your website can be easily extracted if it is accessible to a black cat. So let me guide you in this small tutorial to make sure that your source files are in a safer place where a web user cannot reach it. and the GOOD NEWS is that this tutorial can be implemented on all kind of servers. So lets begin :)

This is how the folders on my web server looks like and it may or need not be the same for every one.

cgi-bin
error-logs
html
	index.html
	.git
temp

All you have to do is move that .git folder out of your html folder and place it in a folder called gitsource or any name of your choice. Each and every web hosting will have a root folder out of which one will be named as “html” “www” “public_html” or any other name where you can store the website files. When you place any files outside that folder it is very difficult to access it. And hence your directory should represent something like this.

cgi-bin
error-logs
gitsource
	.git
html
	index.html
temp

Now that you have moved .git directory out of your website folder, you need to update the git configuration so that whenever you are executing git commands only that particular directory must be affected.

Goto your .git folder and edit the config file in it and append one line that matches the following syntax

[core]
repositoryformatversion = 0
filemode = true
bare = false
logallrefupdates = true
worktree = /home/content/12/12345678/html/

Here we are configuring the git to track files from “/home/content/12/12345678/html” directory. Its a good practise to use an absolute path as i have mentioned it above. This path may vary on different web hosting providers. In order to find your correct path goto your web directory and run the command

$pwd

to print the working directory. Copy paste it to the git config file. In a way your .git folder is in a much secure place now.

You may now find that git throws an error when you run git commands in your website folder as “Not a git repository” – This is completely normal and we know now that its not a git repository since we dont have .git folder.

Then how do you make further commits?

Go to the directory gitsource where we have placed our .git folder and run git commands to make commits. Now, only you can access your git folder and run git commands out of your website folder.
